# 2 Варианты использования

<a name="manage_students"/>

## 2.1 Управление танцорами

**Описание.** Вариант использования "Управление танцорами" позволяет администратору управлять базой данных танцоров.  
**Предусловия.** Администратор авторизован в системе и перешел в раздел "Танцоры".  
**Основной поток.**
1. Приложение отображает список всех танцоров;
2. Администратор выбирает действие: добавить, редактировать или удалить танцора;
3. При выполнении добавления:
   - Администратор вводит данные танцора (ФИО, контакты);
   - Приложение проверяет уникальность данных;
   - Приложение сохраняет нового танцора в базу;
4. При выполнении редактирования:
   - Администратор изменяет данные выбранного танцора;
   - Приложение сохраняет изменения;
5. При выполнении удаления:
   - Администратор подтверждает удаление;
   - Приложение удаляет танцора из базы;
6. Приложение обновляет список танцоров;
7. Вариант использования завершается.

**Альтернативный поток А1.**
1. При попытке добавить существующего танцора приложение выводит сообщение об ошибке;
2. Возврат к п.2 основного потока.

## 2.2 Управление тренерами

**Описание.** Вариант использования "Управление тренерами" позволяет администратору управлять базой данных тренеров.  
**Предусловия.** Администратор авторизован в системе и перешел в раздел "Тренеры".  
**Основной поток.**
1. Приложение отображает список всех тренеров;
2. Администратор выбирает действие: добавить, редактировать или удалить тренера;
3. При выполнении добавления:
   - Администратор вводит данные тренера (ФИО, специализация, контакты);
   - Приложение сохраняет нового тренера в базу;
4. При выполнении редактирования:
   - Администратор изменяет данные выбранного тренера;
   - Приложение сохраняет изменения;
5. При выполнении удаления:
   - Администратор подтверждает удаление;
   - Приложение проверяет, не назначен ли тренер на активные группы;
   - Приложение удаляет тренера из базы;
6. Приложение обновляет список тренеров;
7. Вариант использования завершается.

**Альтернативный поток А2.**
1. При попытке удалить тренера, назначенного на группы, приложение выводит предупреждение;
2. Администратор должен сначала переназначить группы;
3. Возврат к п.2 основного потока.

<a name="manage_groups"/>

## 2.3 Управление группами

**Описание.** Вариант использования "Управление группами" позволяет администратору создавать и редактировать группы.  
**Предусловия.** Администратор авторизован в системе и перешел в раздел "Группы".  
**Основной поток.**
1. Приложение отображает список всех групп;
2. Администратор выбирает действие: создать группу, редактировать состав или удалить группу;
3. При создании группы:
   - Администратор вводит название группы;
   - Выбирает тренера из списка;
   - Добавляет танцоров из базы;
   - Приложение сохраняет новую группу;
4. При редактировании состава:
   - Администратор добавляет или удаляет танцоров из группы;
   - Приложение сохраняет изменения;
5. При удалении группы:
   - Администратор подтверждает удаление;
   - Приложение удаляет группу из базы;
6. Приложение обновляет список групп;
7. Вариант использования завершается.

<a name="view_schedule"/>

## 2.5 Просмотр расписания

**Описание.** Вариант использования "Просмотр расписания" позволяет администратору просматривать расписание занятий.  
**Предусловия.** Администратор авторизован в системе.  
**Основной поток.**
1. Приложение отображает недельную сетку расписания;
2. Администратор может выбрать нужный зал;
3. Администратор может просматривать детали каждого занятия;
4. Вариант использования завершается.

<a name="create_lesson"/>

## 2.6 Создание занятия

**Описание.** Вариант использования "Создание занятия" позволяет администратору добавлять новые занятия в расписание.  
**Предусловия.** Администратор авторизован в системе и находится в разделе расписания.  
**Основной поток.**
1. Администратор выбирает ячейку расписания (время и зал);
2. Приложение отображает форму создания занятия;
3. Администратор выбирает группу и тренера;
4. Приложение проверяет наличие конфликтов:
   - Занятость зала в выбранное время;
   - Занятость тренера в выбранное время;
   - Занятость группы в выбранное время;
5. Если конфликтов нет, приложение сохраняет занятие;
6. Приложение обновляет расписание;
7. Вариант использования завершается.

**Альтернативный поток А3.**
1. При обнаружении конфликта приложение выводит сообщение с деталями;
2. Администратор изменяет параметры занятия;
3. Возврат к п.4 основного потока.

<a name="edit_schedule"/>

## 2.7 Редактирование расписания

**Описание.** Вариант использования "Редактирование расписания" позволяет администратору изменять существующие занятия.  
**Предусловия.** Администратор авторизован в системе и находится в разделе расписания.  
**Основной поток.**
1. Администратор выбирает занятие в расписании;
2. Приложение отображает детали занятия;
3. Администратор выбирает действие: перенести или отменить занятие;
4. При переносе:
   - Администратор выбирает новое время/зал;
   - Приложение проверяет конфликты;
   - Приложение сохраняет изменения;
5. При отмене:
   - Администратор подтверждает отмену;
   - Приложение удаляет занятие из расписания;
6. Приложение обновляет расписание;
7. Вариант использования завершается.
