# Отчет по лабораторной работе: «Улучшение UX танцевального центра»

## 1. Оценка текущего состояния ПО по атрибутам Usability (ISO/IEC 25010:2011)

| Атрибут качества | Краткая оценка текущего состояния («As Is») |
| :--- | :--- |
| **Распознаваемость соответствия** | **Низкая.** Изначально приложение выглядело как стандартная CRUD-система. Пользователь (тренер или студент) видел лишь сухие списки данных без визуального контекста тренировок. |
| **Обучаемость** | **Средняя.** Навигация была функциональной, но неочевидной для новых ролей. Отсутствие персонализированных рабочих областей заставляло пользователей искать нужные функции в общем меню. |
| **Используемость (операбельность)** | **Средняя.** Расписание отображалось списком, что крайне неудобно для планирования. Процесс записи на занятия был перегружен лишними переходами. |
| **Защита от ошибок пользователя** | **Высокая.** Реализована строгая синхронизация ролей и сущностей на уровне базы данных. Ошибки типизации и некорректные приведения типов в контроллерах устранены. |
| **Эстетика GUI** | **Низкая.** Использовались браузерные шрифты по умолчанию, стандартные серые фоны и жесткие границы таблиц. Отсутствовала динамика и современный стиль. |
| **Доступность** | **Средняя.** Интерфейс был адаптивным лишь частично, визуальные акценты в расписании отсутствовали. |

## 2. Выбранные пути улучшения UX и их обоснование

На основе проведенного анализа были выбраны следующие направления рефакторинга интерфейса:

1. **Внедрение современного стиля Glassmorphism и единой дизайн-системы.**
    * *Обоснование:* Использование полупрозрачных панелей, размытия фона и градиентов значительно повышает **Эстетику GUI**. Это создает ощущение премиального продукта и улучшает эмоциональное восприятие пользователем.
2. **Переход к визуальной сетке расписания (Schedule Grid).**
    * *Обоснование:* Вместо списка используется классическая сетка (время/день недели). Это радикально улучшает **Используемость (операбельность)**, позволяя за доли секунды оценить загруженность залов и найти свободное время.
3. **Персонализированные дашборды для каждой роли (Ролевая модель).**
    * *Обоснование:* Студенты и тренеры получают уникальные рабочие области. Это ускоряет **Распознаваемость соответствия** — тренер сразу видит свои группы, а студент — свои занятия.
4. **Ленивое создание и синхронизация профилей.**
    * *Обоснование:* Решает проблемы с **обучаемостью**. Система автоматически создает профиль студента или тренера при первом входе, если у пользователя есть соответствующая роль, исключая ошибки "Missing Data".

## 3. Реализация улучшений (Сравнение «До» и «После»)

### 3.1. Личный кабинет тренера и расписание

**До:** Списки тренеров и групп в виде таблиц в стиле "админки". Отсутствие наглядности. При записи новой группы часто возникали ошибки из-за сложности интерфейса.

**После:** Премиальный дашборд с использованием `backdrop-filter: blur`. Интерактивное расписание с подсветкой занятых слотов. Каждый элемент (карточка группы) содержит стиль танца, уровень сложности и имя тренера.

### 3.2. Техническая стабильность и конгруэнтность данных

**До:** Частые падения Backend'а (`StackOverflowError`) при попытке отобразить вложенные данные в расписании (циклические зависимости Мапперов).

**После:** Оптимизированная архитектура маппинга (Shallow Mapping). Данные загружаются быстро, без ошибок и "битых" связей (N/A в полях).

## Заключение

Проведенный UI/UX рефакторинг превратил технический прототип системы учета в современную платформу для управления танцевальным центром. Сочетание эстетики Glassmorphism и удобной сетки расписания делает продукт конкурентоспособным и приятным в использовании как для администрации, так и для клиентов центра.
